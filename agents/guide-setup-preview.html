Setup : ~20 minutes · Une seule fois

Phase 1 · 15 minutes

Voir les pages en local

VS Code + Live Server

L'objectif : avoir un dossier MGE sur ton Mac avec tous les fichiers du site, et un petit serveur qui recharge automatiquement le navigateur quand tu sauvegardes un fichier.
1

Télécharger le dossier MGE depuis OVH

Ouvre FileZilla et connecte-toi à OVH comme d'habitude.

Dans le panneau de droite (serveur), sélectionne tout le dossier du site. Dans le panneau de gauche (ton Mac), navigue jusqu'au Bureau. Fais glisser le dossier du serveur vers ton Bureau.

Tu as maintenant un dossier MGE sur ton Bureau avec tous les fichiers (HTML, CSS, JS, images).

Tu n'as besoin de faire ça qu'une fois. Ensuite c'est ton dossier de travail local.
2

Installer VS Code

Va sur code.visualstudio.com, télécharge la version Mac, installe comme n'importe quelle application (glisser dans Applications).

code.visualstudio.com →
3

Installer l'extension Live Server

Ouvre VS Code. Dans la colonne de gauche, clique sur l'icône Extensions (4 carrés). Tape Live Server dans la recherche. Clique sur Install sur la première extension (celle de Ritwick Dey, 50M+ téléchargements).

4

Ouvrir le dossier MGE dans VS Code

Dans VS Code : File → Open Folder → sélectionne le dossier MGE sur ton Bureau.

En bas à droite de VS Code, tu vois maintenant un bouton "Go Live". Clique dessus.

Ton navigateur s'ouvre sur localhost:5500 et tu vois ton site exactement comme sur mygreenevent.fr.

C'est tout. Le serveur tourne en arrière-plan tant que VS Code est ouvert. Tu n'as plus rien à faire.
Phase 2 · 10 minutes

Claude écrit les fichiers
directement

MCP Filesystem dans Claude Desktop

L'objectif : que Claude Desktop puisse lire et écrire dans ton dossier MGE. Tu lui demandes une modif, il l'applique directement dans le fichier, Live Server recharge, tu vois le résultat. Zéro copier-coller.
1

Installer Node.js

C'est un outil technique nécessaire pour faire fonctionner le MCP. Va sur nodejs.org, télécharge la version LTS (bouton vert), installe comme une application normale.

nodejs.org →

Pour vérifier que c'est bien installé : ouvre l'application Terminal (cherche "Terminal" dans Spotlight avec Cmd+Espace), tape :

node --version

Si tu vois un numéro de version (ex: v20.11.0), c'est bon.

2

Ouvrir le fichier de configuration de Claude

Dans le Finder, appuie sur Cmd+Shift+G et colle ce chemin :

~/Library/Application Support/Claude

Tu vois un fichier claude_desktop_config.json. Ouvre-le avec VS Code (clic droit → Ouvrir avec → VS Code).

3

Ajouter le MCP Filesystem

Remplace tout le contenu du fichier par ceci (en adaptant le chemin de ton dossier MGE) :

{ "mcpServers": { "filesystem": { "command": "npx", "args": [ "-y", "@modelcontextprotocol/server-filesystem", "/Users/TON-PRENOM/Desktop/MGE" ] } } }
Remplace TON-PRENOM par ton nom d'utilisateur Mac (celui que tu vois dans le Finder avant "Bureau").

Sauvegarde le fichier (Cmd+S).

4

Redémarrer Claude Desktop

Ferme complètement Claude (Cmd+Q), puis réouvre-le.

Dans une conversation, tu peux maintenant dire à Claude : "Lis le fichier index.html dans mon dossier MGE et dis-moi ce que tu vois". S'il répond avec le contenu du fichier, c'est configuré.

En cas de problème

Le site s'affiche sans style dans le navigateur
Tu as probablement ouvert le fichier HTML directement (double-clic) au lieu de passer par Live Server. Ouvre VS Code, clique sur "Go Live", et navigue vers la page dans le navigateur via localhost:5500.
Claude ne peut plus lire/écrire les fichiers
Ferme Claude et rouvre-le. Si ça ne marche toujours pas, contacte Thomas : thomas@sol-ia.tech
Le bouton "Go Live" n'apparaît pas dans VS Code
L'extension Live Server n'est peut-être pas installée. Va dans Extensions (icône à gauche), cherche "Live Server", installe.
