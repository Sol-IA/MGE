Comment tout ça fonctionne

Ton site est un ensemble de fichiers HTML statiques hébergés sur OVH. Pour les petites modifications, tu n'as pas besoin de développeur — tu utilises un assistant Claude dédié, tu vérifies dans ton navigateur, et tu uploades le fichier via FTP.

Le principe en 3 étapes

Tu demandes à Claude de modifier un fichier Ouvre le projet "Site MGE" sur claude.ai. Colle le contenu du fichier HTML à modifier. Décris ce que tu veux changer.
Tu vérifies dans ton navigateur Claude te rend le fichier modifié. Enregistre-le sur ton ordi. Double-clique pour l'ouvrir dans ton navigateur et vérifie que tout est correct.
Tu uploades via FTP Connecte-toi sur OVH avec FileZilla. Glisse le fichier au bon endroit sur le serveur. Le site est mis à jour instantanément.
Pour les modifications importantes (nouvelle section, nouveau design, intégration technique) — contacte Thomas Rodriguez.

Tes assistants Claude

Les 4 agents à ta disposition

Chaque agent est un Projet Claude sur claude.ai. Crée un projet par agent, colle le texte d'instructions correspondant, et utilise-le au quotidien.

Créer un projet Claude : claude.ai → "Projets" → "Nouveau projet" → dans "Instructions du projet", colle le texte du fichier agents/agent-[nom].md correspondant.

Mise en ligne

Checklist avant de remplacer WordPress

Avant d'écraser l'ancien site, s'assurer que tout est prêt. Coche chaque point au fur et à mesure.

Contenu à fournir par Coralie

  • Widget Instagram — code embed de LightWidget (lightwidget.com) avec le compte @mygreenevents
  • Réalisations — photos + infos pour au moins 2-3 mariages (couples, lieu, date, formule, description)
  • Nouvelle clé API Brevo — régénérée depuis le compte et envoyée à Thomas en privé

Technique (Thomas s'en occupe)

  • Fichiers PHP uploadés sur OVH via FTP (contact, newsletter, sécurité)
  • Widget Instagram intégré dans la page d'accueil
  • Portfolio réalisations complété avec les photos

Le jour de la bascule

  • Sauvegarde des photos WordPress téléchargées en local
  • Upload de tous les fichiers du nouveau site via FTP (remplacement complet)
  • Vérification que le site s'affiche correctement sur mobile et desktop
  • Test du formulaire de contact (envoyer un message test)
  • Test du formulaire de newsletter
  • Vérification que les 301 redirections fonctionnent (taper une ancienne URL WordPress)

Dans la semaine qui suit

  • Soumettre le nouveau sitemap dans Google Search Console
  • Vérifier qu'il n'y a pas d'erreurs 404 dans Search Console
  • Supprimer l'installation WordPress depuis le panneau OVH

Ce que les 301 redirections font pour toi

Ton ancien site WordPress avait des URLs comme mygreenevent.fr/mariage-green-faire-part/. Ces URLs sont indexées par Google depuis des années. Le nouveau site a des URLs différentes (/blog/articles/mariage-green-faire-part.html).

Les redirections 301 sont déjà programmées dans le fichier .htaccess du nouveau site. Elles disent à Google : "Cette ancienne page existe maintenant ici." Google transfère automatiquement tout le référencement acquis. Tu ne perds rien.

Résultat : Le SEO construit depuis 2018 — 25+ articles, presse nationale, backlinks — est entièrement préservé après la migration.
