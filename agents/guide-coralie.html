Comment tout ça fonctionne

Ton site est un ensemble de fichiers HTML statiques hébergés sur ton espace OVH. Pour toutes les modifications courantes — textes, prix, articles, réseaux sociaux — tu n'as pas besoin de développeur : tu utilises un assistant Claude dédié, tu vérifies dans ton navigateur, et tu uploades le fichier modifié via FileZilla.

Le principe en 3 étapes

Tu demandes à Claude de modifier un fichier Ouvre le projet correspondant sur claude.ai. Colle le contenu du fichier HTML. Décris ce que tu veux changer. Claude te rend le fichier corrigé, complet.
Tu vérifies dans ton navigateur Enregistre le fichier sur ton ordinateur. Double-clique pour l'ouvrir dans Chrome ou Safari. Vérifie que tout est correct avant de mettre en ligne.
Tu uploades via FileZilla (FTP) Connecte-toi à ton espace OVH avec FileZilla. Glisse le fichier au bon endroit sur le serveur. Le site est mis à jour instantanément.
Pour les modifications importantes — nouvelle section, nouveau design, intégration technique, bug — contacte Thomas : thomas@sol-ia.tech · 06 50 68 20 68

Tes assistants Claude

Les 4 agents à ta disposition

Chaque agent est un Projet Claude sur claude.ai. Une fois créé, tu l'utilises au quotidien sans tout ré-expliquer : il connaît ton site, ta voix, tes offres et tes règles.

Créer un projet : claude.ai → "Projets" → "Nouveau projet" → dans "Instructions du projet", colle le texte du fichier agents/agent-[nom].md correspondant.

Agent "Site MGE" — Modifier les pages

Fichier : agents/agent-site.md

Connaît la structure du site, les couleurs, les règles de navigation. Tu lui colles un fichier HTML + ta demande → il te rend le fichier corrigé complet, prêt à uploader.

Ce qu'il peut faire

  • Modifier un texte, un titre, un prix, un témoignage
  • Ajouter ou mettre à jour une réalisation dans le portfolio
  • Changer une photo (en lui donnant le nom du nouveau fichier image)
  • Créer le fichier HTML d'un nouvel article de blog
  • Mettre à jour blog.html et sitemap.xml en même temps — tu uploades les 3 fichiers d'un coup

Ajouter un article de blog

Workflow complet, de l'idée à la mise en ligne :

Agent Rédaction → article en markdown
→
Agent Site → 3 fichiers HTML
→
FileZilla → upload sur OVH

Étape 1 — Rédaction

Dans l'agent "Rédaction MGE", demande l'article avec le sujet voulu. Il te fournit le texte en markdown + l'entrée XML sitemap.

Étape 2 — Intégration HTML

Dans l'agent "Site MGE", colle le texte de l'article + l'entrée XML sitemap. Il génère 3 fichiers :

  • blog/articles/[slug].html — le fichier de l'article
  • blog.html — la liste mise à jour (nouvelle carte en tête de grille)
  • sitemap.xml — le plan du site mis à jour pour Google

Étape 3 — Upload FileZilla

Connecte-toi à OVH et uploade les 3 fichiers aux bons endroits. L'article est en ligne immédiatement.

Réseaux sociaux : une fois l'article en ligne, ouvre l'agent "Social MGE", colle le texte de l'article → il génère caption Instagram, carrousel et épingle Pinterest en une fois.

Mise en ligne sur OVH

Checklist avant de basculer

Le site est prêt. Avant de remplacer WordPress, vérifier chaque point.

✅ Déjà fait

  • Widget Instagram — Behold.so intégré en page d'accueil (1 200 vues/mois gratuites, HTTPS)
  • Portfolio réalisations — 8 mariages documentés avec photos, galerie lightbox et descriptions SEO
  • Clé API Brevo — insérée dans config.php (newsletter opérationnelle)
  • Toutes les images WordPress — remplacées par des fichiers hébergés en local
  • Redirections 301 — 34 redirections programmées dans .htaccess (SEO préservé)
  • Sitemap.xml — 23 articles + 8 pages principales indexés
  • Pages légales — mentions légales + CGV à jour (avril 2026, SIRET 835 309 006 00012)

À faire le jour J

  • Uploader les fichiers PHP sur OVH via FTP : config.php, send.php, subscribe.php, ratelimit.php
  • Uploader l'ensemble du nouveau site sur OVH (remplacement de WordPress)
  • Tester le formulaire de contact (envoyer un message de test vers contact@mygreenevent.fr)
  • Tester le formulaire newsletter (s'inscrire avec une adresse de test)
  • Vérifier l'affichage sur mobile et desktop
  • Tester une ancienne URL WordPress pour confirmer la redirection 301

Dans la semaine qui suit

  • Soumettre le nouveau sitemap dans Google Search Console
  • Vérifier l'absence d'erreurs 404 dans Search Console
  • Surveiller les erreurs 404 pendant 2–3 semaines
  • Supprimer l'installation WordPress depuis le panneau OVH (seulement après validation complète)

Ce que les redirections font pour toi

Ton ancien site WordPress avait des URLs comme mygreenevent.fr/mariage-green-faire-part/ indexées par Google depuis des années. Le nouveau site utilise /blog/articles/mariage-green-faire-part.html.

Les 34 redirections 301 déjà programmées dans .htaccess disent à Google : "Cette ancienne page existe maintenant ici." Google transfère automatiquement tout le référencement.

Résultat : Les 23 articles de blog, les backlinks de la presse nationale (Figaro, France Info, RTL, M6) et le référencement construit depuis 2018 sont entièrement préservés après la migration.
